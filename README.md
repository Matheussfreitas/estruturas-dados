# Estruturas de Dados e Algoritmos

## Orientação e objetivo

Este repositório organiza estudos práticos de estruturas de dados e algoritmos.
O foco é construir uma base técnica e analítica para entender como as coisas
funcionam, com exercícios e implementações simples.

Objetivos principais:

- intuição de escalabilidade (Big-O)
- habilidade lógica de programação
- melhor resolução de problemas
- habilidade agnóstica a linguagens
- confiança para entender códigos
- avançar na carreira
- preparo para entrevistas

## Como navegar

Cada pasta representa um módulo. Há arquivos de apoio em Excalidraw e
implementações em JS/TS com exemplos clássicos.

Sugestão de fluxo:

1. Big-O e análise de complexidade
2. Arrays e técnicas básicas
3. Sorting
4. Linked List
5. Stack
6. Binary e bitwise
7. Binary Tree
8. Graphs
9. Heap
10. LeetCode (exercícios aplicados)

## Módulos e o que estudar

### bigO-notation/

Base de análise de complexidade com diagramas para:

- O(1)
- O(log N)
- O(N)
- O(N log N)
- O(N^2)

### arrays/

Fundamentos de arrays e técnicas comuns:

- two-pointer
- sliding window
- binary search
- exponential search

### sorting/

Algoritmos de ordenação com exemplos:

- bubble sort
- merge sort
- quick sort

### linked-list/

Listas ligadas e operações centrais:

- reverse linked list
- middle of linked list
- cycle detection
- doubly linked list

### stack/

Estruturas LIFO e problemas clássicos com pilhas.

### binary/

Operações bitwise e exercícios:

- and/or/not/xor
- shift left/right
- missing number com XOR
- reduce numbers to zero

### binary-tree/

Construção e busca em árvores binárias:

- build binary tree
- BFS e DFS
- search binary tree
- variações estilo LeetCode

### graphs/

Grafos e algoritmos:

- dijkstra
- clone graph
- course schedule
- network delay time
- grafos direcionados

### heap/

Estruturas de heap e uso em problemas de prioridade.

### leetcode/

Aplicação prática dos módulos em JS/TS:

- arrays (binary search, sliding window, two-sum, etc.)
- linked-list (reverse linked list)

## Como usar

- Leia o diagrama Excalidraw quando existir.
- Estude o código JS/TS correspondente.
- Reescreva a solução com outra abordagem e compare complexidade.
